# Security Policy

## Reporting A Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, contact the maintainer privately through GitHub or another private channel listed by the repository owner. Include:

- a description of the issue
- steps to reproduce it
- affected environments or configuration details
- any suggested mitigation if you have one

## Scope

Relevant reports include issues such as:

- unauthorized access paths
- secret handling problems
- unsafe local machine operations
- dependency vulnerabilities with a clear impact on this project

General support questions and feature requests should go through normal project discussions or issues.

## Response Expectations

- Responses are best-effort.
- There is no guaranteed SLA for triage or remediation.
- Please avoid public disclosure until the maintainer has had a reasonable chance to review the report.
